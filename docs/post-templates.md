# Post Templates

이 문서는 글을 대신 써주는 문서가 아니라, 직접 쓰기 위한 빈 틀입니다.
문체, 사례, 판단, 아쉬움은 작성자가 직접 채우고, 아래 질문은 글이 기술 블로그답게 흐르도록 잡아주는 체크리스트로만 사용합니다.

## 사용 원칙

- 첫 문단에는 이 글을 왜 쓰는지 적습니다.
- 기술 이름만 나열하지 말고, 선택 이유와 포기한 대안을 같이 적습니다.
- 성공한 부분뿐 아니라 막힌 점과 아쉬운 점을 남깁니다.
- 마지막에는 다음에 개선할 방향을 적습니다.
- 글 하나는 길게 쓰기보다, 한 문제를 끝까지 설명하는 데 집중합니다.

```md
## 왜 만들었나?

- 어떤 문제를 보고 시작했는가?
- 기존 방식에서 불편했던 점은 무엇인가?
- 이 프로젝트로 확인하고 싶었던 것은 무엇인가?

## 전체 구조

- 주요 구성 요소는 무엇인가?
- 프론트엔드, 백엔드, 인프라는 어떻게 나뉘는가?
- 데이터나 요청 흐름은 어떻게 이동하는가?

## 구현하면서 중요했던 선택

- 왜 이 기술을 골랐는가?
- 대안은 무엇이었고, 왜 제외했는가?
- 가장 오래 고민한 부분은 무엇인가?

## 막혔던 점

- 어디서 막혔는가?
- 원인은 무엇이었는가?
- 어떻게 해결했는가?

## 결과와 아쉬운 점

- 지금 동작하는 것은 무엇인가?
- 아직 부족한 부분은 무엇인가?
- 다음에 개선한다면 무엇부터 할 것인가?
```

## 기존 글 다듬기 체크리스트

- 제목이 글의 결론이나 질문을 분명하게 보여주는가?
- 첫 문단만 읽어도 왜 이 글을 쓰는지 알 수 있는가?
- 장점과 단점이 모두 들어 있는가?
- 내 경험에서 나온 판단이 들어 있는가?
- 마지막 문단에 다음 글이나 다음 개선 방향이 자연스럽게 이어지는가?
